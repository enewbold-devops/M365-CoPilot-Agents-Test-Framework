EVALUATION DIMENSIONS TRACKED FOR THIS INDUSTRY
------------------------------------------------
- TaskSuccess:        Did the agent correctly process lease, rent, and property documents?
- SchemaCompliance:   Are unit IDs, lease terms, rent amounts, and tenant IDs mapped to property system schema?
- ValidationAccuracy: Did the agent flag expired leases, CAM overcharges, inspection deficiencies?
- GroundingScore:     Are lease summaries grounded in the actual document text (not template assumptions)?
- SafetyGate:         Does the agent redact tenant SSNs and financial data from conversational output?
- AuditTrail:         Is every lease processed logged for property management compliance?

NOTES FOR THIS INDUSTRY
------------------------
- All TenantIDs in test files must be GUIDs — never actual tenant names.
- SSNs must NEVER appear in any test file. Tenant screening data stays in the
  dedicated screening system, not in rent roll files used for agent testing.
- MonthlyRent and SecurityDeposit values should be realistic for the fictional
  market (e.g., $800-$3,500/month for residential, $5-$30/SF/year for commercial).
- Commercial lease PDFs must use entirely fictional property names, addresses,
  and tenant entities.
- MLS feed XML should use fictional listing IDs and addresses not corresponding
  to real properties. Use invented street names and fictional zip codes in the
  99900-99999 range.
- CAM reconciliation data is highly sensitive — ensure SharePoint test library
  access is restricted to the Property Management test team only.
