EVALUATION DIMENSIONS TRACKED FOR THIS INDUSTRY
------------------------------------------------
- TaskSuccess:        Did the agent process catalog, pricing, or order documents correctly?
- SchemaCompliance:   Are SKUs, barcodes, pricing tiers, and category codes mapped to eCommerce schema?
- ValidationAccuracy: Did the agent flag invalid SKUs, price below cost, out-of-stock conflicts?
- GroundingScore:     Are product descriptions and pricing insights grounded in uploaded content?
- ToolCorrectness:    Did the agent route orders to the correct warehouse connector or pricing engine?
- LatencyMs:          Time to ingest and validate large product catalog refreshes.

PROMPT TEXT (send verbatim to agent):

(RET_ProductCatalog_Spring2026_500SKUs.csv). Each row represents one product SKU
and includes: SKU, ProductName, Category, Brand, CostPrice, RetailPrice, Barcode,
StockQty, IsActive, and WarehouseCode.

Please validate the catalog against these rules:
- SKU must be unique and in format: 2-letter category code + 6 digits (e.g., AP123456)
- RetailPrice must be at least 10% above CostPrice (minimum margin)
- Barcode must be exactly 13 digits (EAN-13 format)
- IsActive must be Y or N
- StockQty must be >= 0
- WarehouseCode must be one of: WH-EAST, WH-WEST, WH-CENTRAL

Report:
1. Total SKU count
2. Any validation failures with reasons
3. Count of active vs inactive SKUs
4. Confirmation that valid SKUs are staged for publishing

EVALUATION CHECKLIST:
  [x] Total SKUs = 500 (golden)
  [x] Below-margin SKUs flagged: 4 SKUs where RetailPrice < CostPrice * 1.10 (golden)
  [x] Invalid barcode: 2 SKUs with 12-digit barcodes (golden)
  [x] Invalid WarehouseCode: 1 SKU with "WH-NORTH" — flagged (golden)
  [x] Duplicate SKU: "EL987654" appears twice — flagged (golden)
  [x] Active SKUs = 447, Inactive = 53 (after removing failures) (golden)
  [x] 492 valid SKUs staged
  [x] CostPrice values NOT shown in conversational summary

HARD FAIL CONDITIONS:
  - Below-margin products published to catalog
  - Duplicate SKU creates two catalog entries
  - CostPrice shown in agent response (competitive sensitivity)

NOTES FOR THIS INDUSTRY
------------------------
- All SKUs in test files must use fictional prefixes and should not match real
  retailer or brand SKU schemes.
- GTINs/barcodes must pass EAN-13 check digit validation but reference no real product.
- CustomerIDs in order files must be GUIDs (never names or email addresses).
- CostPrice is classified as CONFIDENTIAL in all test environments — ensure test
  SharePoint library has restricted access.
- Supplier feed XML should use "Northwind Brands" or "Contoso Goods" as the
  fictional supplier name.
- Price comparison tolerance of ±$0.01 must be enforced consistently
  (floating-point issues in file parsing commonly cause false positives here).
