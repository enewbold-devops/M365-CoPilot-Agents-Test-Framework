EVALUATION DIMENSIONS TRACKED FOR THIS INDUSTRY
------------------------------------------------
- TaskSuccess:        Did the agent correctly process supplier, inventory, or logistics documents?
- SchemaCompliance:   Are part numbers, quantities, UOM, and vendor codes mapped to the ERP schema?
- ValidationAccuracy: Did the agent flag discrepancies — quantity mismatches, invalid part numbers, lead time violations?
- GroundingScore:     Are supply chain insights derived from the document, not hallucinated?
- ToolCorrectness:    Did the agent invoke the correct downstream connector (ERP staging, supplier portal)?
- LatencyMs:          Time to process and confirm large BOM or PO batches.

NOTES FOR THIS INDUSTRY
------------------------
- Part numbers in all test files must use fictional prefixes ("AB", "CD", "EF") and
  should not replicate real ISO or ANSI part numbering from real manufacturers.
- EDI 850 XML test file should loosely follow X12 element naming but does not require
  a fully compliant X12 envelope — simplified XML representation is acceptable for
  agent testing purposes.
- ERP staging table references should use fictional system names (e.g., "ContosoERP")
  in all test prompts and agent response validation.
- Supplier names in all test files: use "Supplier ABC", "Supplier XYZ", "Northwind
  Components" — never real company names.
